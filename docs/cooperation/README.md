## 团队角色

| 角色           | 职责                                   | 技术栈                     |
| -------------- | -------------------------------------- | -------------------------- |
| **后端工程师** | API 设计与实现、折叠服务集成、数据模型 | Python, FastAPI, Redis     |
| **前端工程师** | UI 组件开发、Figma 设计还原、交互实现  | React, Next.js, TypeScript |

## 核心原则

1. **API 先行**: 先定义接口契约，再各自实现
2. **Mock 优先**: 前端使用 Mock 数据独立开发，后端提供 Mock 模式
3. **增量联调**: 功能模块完成后立即联调，避免积压
4. **文档驱动**: 接口变更必须更新文档
